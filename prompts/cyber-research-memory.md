# Cyber Research Memory Loop

Use cybermem as a bounded, auditable research memory.

At task start:

1. Set the `goal` buffer with objective, target, scope, and constraints.
2. Recall by combining the vulnerability description, project, sanitizer, file format, and current blocker.
3. Read the retrieval trace if results look surprising.

During work:

1. Keep `tool` and `imaginal` buffers short and current.
2. Recall again when the bug class, subsystem, or failure mode changes.
3. Link memories when a prior lesson explains a current observation.

At task end:

1. Remember what worked, what failed, exact evidence, and why the result is in scope.
2. Tag memories with project, format, sanitizer, bug class, and outcome.
3. Consolidate repeated lessons into procedures or principles.

Do not store secrets, raw credentials, or benchmark-specific seeded task data.
